# 项目技术文档中心

> **项目**: OAuth2.1 认证授权中心 & 微服务平台
> **文档版本**: v3.0.0  
> **最后更新**: 2025-06-27
> **维护团队**: 全体开发团队

## 文档概述

本文档库包含基于 Turborepo 的 OAuth2.1 认证授权中心和微服务平台的核心技术文档。项目采用 Next.js 15 + TypeScript 5 技术栈，实现了完整的 OAuth2.1 + OIDC 认证授权体系和 RBAC 权限管理系统。所有文档均采用中文编写，遵循统一的格式规范。

## 项目技术栈

- **框架**: Next.js 15.3.2 (App Router)
- **语言**: TypeScript 5
- **UI**: shadcn/ui + Tailwind CSS 4.1.5
- **数据库**: Prisma 6.10.1 + SQLite
- **认证**: OAuth2.1 + JWT (jose 6.0.11)
- **构建工具**: Turborepo 2.5.4
- **包管理**: pnpm 10.6.2

## 微服务架构

本项目包含 5 个独立的微服务应用：

| 服务名称          | 端口 | 描述                  | 技术特色                   |
| ----------------- | ---- | --------------------- | -------------------------- |
| **oauth-service** | 3001 | OAuth2.1 认证授权服务 | JWT + PKCE + RBAC          |
| **admin-portal**  | 3002 | 系统管理后台          | shadcn/ui + TanStack Table |
| **kline-service** | 3003 | K线图表服务           | WebAssembly + Rust         |
| **flow-service**  | 3004 | 可视化流程引擎        | React Flow + Zustand       |
| **test-service**  | 3005 | 测试与示例服务        | OAuth2.1 客户端示例        |

## 快速导航

### 🚀 新手入门

1. **了解项目背景**: 阅读 [产品需求文档](./产品需求文档.md) 了解 OAuth2.1 认证授权中心的业务需求
2. **理解系统架构**: 查看 [系统架构设计](./系统架构设计.md) 掌握 Turborepo 微服务架构
3. **深入技术细节**: 研究 [详细设计文档](./详细设计文档.md) 了解数据模型和 API 设计
4. **快速启动**: 运行 `pnpm dev` 启动所有微服务进行本地开发

### 👨‍💻 开发人员

- **认证开发**: 遵循 [JWT认证授权使用说明](./JWT认证授权使用说明.md) 实现 OAuth2.1 集成
- **API开发**: 参考 [详细设计文档](./详细设计文档.md) 中的 API 规范
- **测试开发**: 按照 [测试策略文档](./测试策略文档.md) 编写 Jest 测试用例
- **权限控制**: 使用 RBAC 服务进行细粒度权限管理

### 🎨 设计人员

- **UI设计**: 参考 [管理页面设计](./管理页面设计.md) 进行 admin-portal 界面设计
- **组件库**: 使用 `@repo/ui` 共享组件库保持设计一致性

### 📝 文档维护

- **规范遵循**: 所有文档修改都应遵循 [文档规范指南](./文档规范指南.md)
- **版本管理**: 文档与代码同步更新，保持技术文档的时效性

## 贡献指南

### 开发环境搭建

```bash
# 安装依赖
pnpm install

# 启动所有微服务
pnpm dev

# 数据库初始化
pnpm db:push
pnpm db:seed

# 运行测试
pnpm test
```
