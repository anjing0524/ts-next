## 📈 文档维护指南

### 文档命名约定

- **00-** 前缀: 当前重要文档 (评估、总结、索引)
- **01-** 前缀: 之前版本或参考文档
- **数字前缀** (1-7): 核心设计文档
- **无前缀**: 专题分析文档

### 更新规则

| 情况 | 做法 |
|------|------|
| 修复重大问题 | 创建新的评估/修复文档，包含日期 |
| 日常更新 | 在现有文档中更新内容 |
| 归档旧文档 | 重命名为 **01-** 或更高前缀 |
| 创建新专题 | 用清晰的英文名称，避免使用日期 |

### 版本控制

所有重要文档应包含：
- **最后更新时间**: YYYY-MM-DD
- **评估者/作者**: 名字或团队
- **状态**: ✅ (当前) / 📋 (参考) / 📚 (历史)
